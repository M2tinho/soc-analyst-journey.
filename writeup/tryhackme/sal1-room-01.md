SAL1 — Room 01 : SOC Fundamentals

What is a SOC?

A Security Operations Center is a team responsible for monitoring, detecting, analyzing, and responding to cybersecurity incidents — continuously, 24/7.

Key distinction: a SOC is internal to a company. An MSSP (Managed Security Services Provider) is the external equivalent — a specialized third party that delivers SOC services to multiple clients.

SOC vs MSSP

SOC	MSSP
Structure	Internal team	External provider
Clients	One company	Multiple clients
Variety	Consistent environment	Different tools per client
Best for	Depth in one context	Breadth and polyvalence fast
My take: MSSP is interesting for juniors specifically because you're constantly exposed to new environments, new tools, new client setups. It forces polyvalence faster than a single internal SOC would.

Career path

SOC L1 Analyst ↓ SOC L2 / SOC Engineer ↓ CIRT (Computer Incident Response Team) ↓ CISO

What changes at CIRT level: you can no longer depend on SIEM/EDR tools to guide your thinking. You need broad threat knowledge and the ability to handle breaches from first principles.

Narrow specialisation roles worth knowing

Role	Core skill
Digital Forensics Analyst	Uncover hidden threats in disk and memory
Threat Intelligence Analyst	Track and profile emerging threat groups
AppSec Engineer	Maintain a secure software development lifecycle
AI Security Researcher	Study AI-specific threats and defenses
Key takeaway

The L1 role is not the destination — it's the foundation. The goal is to build pattern recognition on real alerts so that moving to L2 or CIRT is a natural progression, not a jump.
